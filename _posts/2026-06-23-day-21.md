what_i_learned: |
  Today was a continuation of the individual model training task that we began
  earlier this week. I successfully trained a model on my cleaned dataset, but
  the results were not as promising as I had hoped. Instead of immediately
  retraining the model, I spent time debugging the pipeline to identify
  potential issues.

  I carefully checked that images were being loaded and processed correctly,
  verified that the data was flowing through the pipeline as expected, and
  reviewed the model architecture to ensure it matched our intended design.
  This process helped me better understand how many components must work
  together for a machine learning model to perform well.

  Later in the day, our team shifted focus to the application component of our
  project. We began setting up the requirements and dependencies needed to run
  the skin lesion diagnosis application that our mentor has been developing.
  Tomorrow, we plan to start running the application locally and making changes
  to support our project goals.

  We also took a break from coding and participated in a group game of
  Imposter. It was a fun way to relax, recharge, and spend time with other
  participants before returning to our technical work.

blockers: |
  My biggest challenge today was setting up the project's GitHub repository in
  VS Code. Many of the setup instructions provided were written for macOS,
  while I am working on a Windows machine. This created several configuration
  issues that made it difficult to get everything running correctly.

  I am still experiencing some setup problems and hope to resolve them
  tomorrow so that I can fully contribute to the application development work.

reflection: |
  Today reminded me that a large part of machine learning involves debugging
  and troubleshooting rather than simply training models. When results are not
  what you expect, it is important to investigate the data pipeline, model
  architecture, and preprocessing steps before making changes.

  Although the model performance was disappointing, I learned a lot from the
  debugging process and gained a deeper understanding of how machine learning
  systems operate behind the scenes.

  I am also excited that we are beginning work on the application itself. After
  spending weeks focused on data preparation, literature reviews, and model
  training, it feels rewarding to start working on the software that will bring
  our research together into a usable tool.
